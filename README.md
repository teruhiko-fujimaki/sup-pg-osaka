

# 概要仕様書：特別支援教育「すけっと」連携型・計画作成支援アプリ (Demo版)

## 1. アプリの目的・コンセプト

* **目的:** 小学校の知的障害特別支援学級担任（特に経験の浅い教員）の授業づくりと計画作成を、「すけっと（助っ人）」として支援する。
* **コンセプト:** 「実態把握（アセスメント）」から「指導目標（スモールステップ）」、そして「本時の授業設計」までを一本の線で繋ぎ、児童の「できた！」という笑顔を引き出す。

## 2. 画面設計・UI仕様

* **キャンバス幅:** 1600px（PCでの作業効率を最大化）
* **基本レイアウト:**
* **左パネル (400px):** 児童の基本情報・特性・「すけっと」専門家ガイド（常時表示）。
* **メインエリア (1200px):** 計画作成エディタ。タブ切り替えで「教育支援計画」「指導計画」「授業サポートシート」を編集。


* **デザイン:** Antigravityの標準コンポーネントをベースとしたカード型UI。

---

## 3. 主要機能モジュール

### 3.1 児童実態・ニーズ管理

* **実態把握入力:** 日常生活、学習、行動等の6領域を入力。
* **リフレーミング機能:** 課題（例：衝動的）を前向きな特性（例：表現意欲が高い）にAIが自動変換する。
* **関係機関ネットワーク:** 医療・福祉・地域との連携リストの管理（大阪府様式準拠）。

### 3.2 「指導内容ステップ表」連動・目標生成

* **AI目標生成:** 入力された実態から、NISEの知見に基づいた「A・B・C」の3段階（スモールステップ）の短期目標を提案する。
* **ステップC:** 基礎的な受容（わかる、選ぶ）。
* **ステップB:** 具体物を用いた活動（〜を使って測る、〜で伝える）。
* **ステップA:** 抽象的・自立的な活動（定規で測る、言葉で交渉する）。


* **教科別アドバイス:** 国語科・算数科の「指導のコツ」をAIがレコメンドする（例：具体的から抽象的へ）。

### 3.3 NISE授業づくりサポートシート生成

* **一貫性保持:** 指導計画の目標を、即座に「本時の学習活動」のドラフトへ変換する。
* **手だて・教具提案:** 実態（例：聴覚過敏）に応じた具体的な配慮（例：イヤーマフの使用、視覚支援カード）を提案する。

## 5. デモ版の運用フロー

1. **実態の言語化:** 先生が日々の観察をラフに入力する。
2. **AI支援:** AIが「すけっと」の視点で実態を専門的な文章に整え、3段階の目標案を提示する。
3. **授業への落とし込み:** 目標に合わせた「本時の展開」と「必要な教材・教具（10cmブロック、写真カード等）」を生成する。
4. **プレビュー・調整:** 1600pxの画面で全体の一貫性を確認し、必要に応じて微調整して確定する。
